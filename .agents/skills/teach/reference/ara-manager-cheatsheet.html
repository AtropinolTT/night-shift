🗂️ /ara-manager — Quick Reference

All commands at a glance. Your ARA project lives at ./ara/.

👀 Inspect (Read-only)

status
Dashboard — claims, staged obs, tree stats, open threads
/ara-manager status --json
briefing
Full sitrep — last session, open threads, stale obs, claims
Good first command each session
tree
Print exploration DAG (decisions, experiments, etc.)
tree --node N12 --depth 3 --type experiment

✏️ Record (Write)

log decision
A deliberate choice between alternatives
log decision "Use X over Y" --provenance user
log experiment
Experiment run + outcome (even negative)
log experiment "R11 HPO: params..." --provenance ai-executed
log dead_end
A path that didn't work
log dead_end "PDI R²≈0 — artifact" --provenance user
log pivot
Change in direction, with trigger
log pivot "RF → XGBoost" --provenance user
log question
An open research question
log question "Does uncertainty help AL?" --provenance ai-suggested
add-claim
Stage a falsifiable assertion
add-claim "EE R²=0.93" --bound-to N30,N45

🔄 Advance (Promote)

crystallize
Promote staged O{XX} into logic/
crystallize O114 --via resolution
advance-claim
Update C{XX} status (hypothesis → testing → supported)
advance-claim C07 supported --note "54 tests pass"
revise-claim
Rewrite statement/rationale of C{XX}
revise-claim C07 --statement "..." --rationale "..."

🤖 Automated (Batch)

update with docs/
Scan files for claims + decisions → stage
update with docs/handoffs/
update claims
Suggest which claims can advance status
Read-only; confirms before changing
update obs
Suggest which obs can crystallize
Read-only; confirms before promoting
compile
3-pass deep read + stage + optional crystallize
compile docs/ --crystallize

🛡️ Quality

validate
Seal Level 1 structural checks (17 checks)
Cross-layer refs, provenance, ID uniqueness
review
Seal Level 2 epistemic audit (delegates to rigor-reviewer)
For deep claim-evidence audit

🏗️ Init

/ara-manager init "Project Title" --domain "..." --keywords k1,k2 --abstract "..."
Already done for MARLIN — only needed for new projects.

Closure Signals

SignalUse whenEffect
affirmationUser explicitly endorsed itUpgrades provenance
resolutionExperiment completed, user commentedStandard promote
abandonmentNo events for 5+ turnsClean up stale obs
commitmentDownstream artifact depends on itCode/config merged

Provenance Values

ValueMeaning
userYou said it / made the call
ai-suggestedClaude proposed it, not yet confirmed (🥇 safe default)
ai-executedClaude ran the experiment / generated the result
user-revisedHybrid — Claude proposed, you edited

Claim Status Transitions

hypothesis → testing → supported
hypothesis → testing → weakened → (stays weakened)
hypothesis → refuted (terminal — must revise-claim to revive)
any → withdrawn (terminal, non-empirical)
any → revised (transition marker, resets to hypothesis/testing)
Habit: Start each session with briefing → tree --type question. End each session with log experiment "...".
