🧬 BCR Tree + Flow Matching Glossary

A quick reference — open each term to expand. Tagged by domain: bio math ML code

AbLang2 ML
An antibody-specific protein language model. Takes an antibody sequence and produces a 2048-dimensional embedding vector per sequence. Pre-trained on millions of antibody sequences. Used in our project to encode BCR sequences — the embedding captures functional properties (antigen binding, stability). Key: In our model, the guidance MLP predicts velocity in AbLang2 embedding space.
See also: Guidance MLP
Affinity maturation bio
The process by which B cells evolve to produce antibodies with higher antigen-binding affinity. Occurs in germinal centers. Key mechanisms: SHM (creates diversity) + selection (survival of high-affinity clones). Our model aims to simulate this — Mode B (antigen-guided evolution) with β>0 should produce sequences that move toward a target antigen's embedding.
See also: SHM
Balanced tree bio
A phylogenetic tree where branching events are spread across the tree, with multiple internal nodes representing shared ancestors. In BCR context: biologically realistic — some B cells divide and mutate more than others, creating a hierarchical structure. Contrast: star trees (all leaves from one ancestor), which are often artifacts. After Bug #2 fix, our trees went from mean 1.0 to mean 15.1 internal nodes.
See also: Star tree
Benjamini-Hochberg (BH) adjustment math
A method to control the false discovery rate (FDR) when testing multiple hypotheses simultaneously. Used in Q4 evaluation to correct p-values across all tested features. Threshold: q < 0.05 (expected ≤5% of "discoveries" are false positives).
CDR3 bio
Complementarity-Determining Region 3 — the most hypervariable region of the BCR. Formed by V-(D)-J recombination. The primary determinant of antigen specificity. In our data: clones are defined by shared V gene, J gene, and CDR3 length (plus CDR3 similarity). CDR3 is the main target of SHM.
Clone / Clonal family bio
A group of B cells that descended from the same naive B cell — sharing the same V(D)J rearrangement. In our project: 281 clones from healthy donors, each clone producing a lineage tree. Clone threshold is set empirically via silhouette-score-based optimization (though the code warns this is the wrong criterion for clone definition).
Conservative substitution rate bio
The fraction of mutations where the new amino acid is biochemically similar to the old one (BLOSUM62 score > 0). Real SHM heavily favors conservative substitutions (rate ~0.361 in our data). Random mutations would be ~0.124. Our model went from 0.124 → 0.232 after full training — improving but not yet matching biology. Key metric for evaluating whether the model learns real SHM patterns beyond just being "valid."
See also: SHM
CTMC (Continuous-Time Markov Chain) math
A Markov model where: (1) you're always in one of N discrete states (20 AAs), (2) you can jump between states at any time, (3) waiting times are exponentially distributed (memoryless), (4) transitions are governed by a rate matrix Q. Key properties: rows of Q sum to zero; the diagonal is negative (total exit rate). Transition probability over time t: P(t) = exp(Q × t) (matrix exponential). Training: sample random t, maximize P(parent → child | Q). Our model uses one CTMC per sequence position.
See also: Matrix exponential, Rate matrix
Coupling formula math
The formula that connects the discrete CTMC core to the continuous guidance MLP: score = R[i→j] × exp(β · cos_sim(v, e_child - e_parent)). When β=0, guidance = 1, and we use pure CTMC (Mode A). When β>0, the velocity field v modulates the rates — speeding up mutations that align with the guidance direction. Intuition: The CTMC says "S→G is biochemically likely (rate 2.0)." The guidance says "moving toward this antigen embedding requires S→G." The coupling says "then S→G is doubly good — both likely and functional."
See also: CTMC, Guidance MLP
Flow matching ML
A generative framework where a model learns to transform a simple distribution into a target distribution by learning a velocity field (continuous) or transition rate matrix (discrete). Key difference from diffusion: flow matching solves an ODE (deterministic given initial condition) while diffusion solves an SDE (stochastic). This makes flow matching potentially more efficient at inference. In our project: hybrid approach — discrete CTMC for amino acids + continuous guidance in embedding space.
See also: CTMC
Guidance MLP ML
A 3-layer MLP (2048→512→256→512→2048) that predicts velocity in AbLang2 embedding space. Input: interpolated embedding e_t = (1-t)×e_parent + t×e_child. Output: predicted velocity v. Loss: MSE(v_pred, e_child - e_parent). Current status: training but guidance loss ~0 because set_embeddings() was never called — embeddings are all zeros. Guidance MLP has learned "predict zero velocity" which is optimal for zero inputs.
See also: AbLang2, Coupling formula
Matrix exponential math
NOT element-wise exp! Defined as the infinite series: exp(A) = I + A + A²/2! + A³/3! + ... (like the regular exponential series, but with matrix multiplication). Solves the ODE dP/dt = P × Q. Key property: if rows of Q sum to zero, rows of exp(Q×t) always sum to 1 (valid probabilities). For t→0: exp(Q×t) ≈ I + Q×t. For t→∞: converges to equilibrium distribution. Computing this is expensive (O(n³) for n=20) but 20×20 matrices are small enough. In PyTorch: torch.linalg.matrix_exp().
See also: CTMC, Rate matrix
NJ (Neighbor-Joining) tree bio
A distance-based phylogenetic tree building method. Fast but less accurate than ML (maximum likelihood). In our project: used to build AA-level trees from Hamming distances between CDR3 sequences. Known limitation: NJ tends to produce star trees when sequences are short or highly similar (exactly our case — CDR3s are short). This is a key source of uncertainty: 53% star trees may reflect NJ's limitations, not real biology.
See also: Balanced tree, Star tree
Rate matrix Q math
The heart of a CTMC. A 20×20 matrix where Q[i][j] (i≠j) = instantaneous rate of transition from state i to state j. Q[i][i] = -sum(Q[i][j] for j≠i) — so each row sums to zero. Units: expected transitions per unit time. If Q[S→G] = 2.0, then in a tiny time dt, expected transitions from S to G = 2.0 × dt. Waiting time for any specific transition is exponentially distributed with rate Q[i][j]. Our model predicts one Q per sequence position (because CDR and framework positions mutate differently).
See also: CTMC, Matrix exponential
Regression to the mean ML
The tendency of probabilistic models to predict the "average" outcome rather than the full distribution. In our model: generated sequences have 17.6 ± 1.8 mutations vs real 26.5 ± 39.2. The model plays it safe — it learns the average mutation pattern and produces plausible-looking, 100% valid sequences that lack the diversity of real SHM. Why it matters: high validity ≠ good model. A model that always predicts the mean won't generate rare but functionally important mutations. Combatting this is an active research area (temperature scaling, top-k sampling, etc.).
SHM (Somatic Hypermutation) bio
A process where activated B cells introduce point mutations into their BCR genes at a rate ~10⁶× higher than the normal genomic mutation rate. Mediated by the enzyme AID. Targets the V(D)J region (~400 bp), with hotspots in CDRs. Key properties: (1) primarily point mutations (indels rare), (2) favors conservative substitutions (BLOSUM62 > 0), (3) higher rate in CDRs than framework, (4) occurs in germinal centers, (5) combined with selection → affinity maturation. Our model learns the "grammar" of SHM from tree edges.
See also: Affinity maturation, Conservative rate
Spearman's rank correlation (rho) math
A non-parametric measure of monotonic association between two variables. Unlike Pearson: doesn't assume linearity, works with ranks. In Q4 evaluation: used to correlate tree features (e.g., stemminess, depth) with functional outcomes (antigen binding). Thresholds: r_min = 0.125 (80% power detection floor at n=499) and min_spearman_rho = 0.3 (biological significance). Both must pass for "publication-ready" verdict.
See also: BH adjustment
Star tree bio
A phylogenetic tree where all leaves connect directly to a single ancestor — no branching structure. Looks like a starfish. In BCR context: usually an artifact of poor tree inference (NJ with short sequences), not real biology. Our data had 53% star trees before Bug #2 fix — and actually 100% due to the empty-string key bug (all internal nodes collapsed). After fix: mean internal nodes went from 1.0 → 15.1. Star trees provide no topological information — you can't compute meaningful features like depth, stemminess, or balance.
See also: Balanced tree, NJ tree
